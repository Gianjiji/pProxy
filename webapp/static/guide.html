Come anonimizzare i dati sensibili prima di darli a un'AI e ripristinarli nella risposta.

A cosa serve

pProxy sostituisce i dati sensibili (email, telefoni, IBAN, codici fiscali, nomi, indirizzi…) con segnaposto tipo [EMAIL_001]. Così puoi mandare il testo a un'AI senza esporre i dati reali; quando l'AI risponde (mantenendo i segnaposto), pProxy ricostruisce gli originali. La mappa segnaposto→valore reale è il dato più delicato: non viene mai inviata all'AI né scritta in chiaro.

I due modi di usarlo

A) Flusso manuale — usi la TUA AI (es. ChatGPT, Claude nel browser)

  1. Anonimizza (scheda 1): incolla il testo → Anonimizza. Ottieni il testo con i segnaposto e un ID sessione. Premi Copia.
  2. Incolla quel testo nella tua AI e fatti dare la risposta. Importante: chiedi all'AI di non modificare i segnaposto […_NNN].
  3. Ripristina (scheda 2): incolla la risposta dell'AI → Ripristina dati originali. L'ID sessione è già compilato. Ottieni il testo con i dati reali.

È il flusso che preserva di più la privacy: i dati reali non lasciano la tua macchina/il server verso l'AI. Con Zero-knowledge la mappa resta solo nel tuo browser (vedi sotto).

B) Pipeline automatica — pProxy chiama l'AI al posto tuo

  1. Pipeline LLM (scheda 3): incolla il testo o carica un file, scegli il provider.
  2. Premi Esegui pipeline: pProxy anonimizza, invia all'AI (con la chiave API configurata sul server) e ti restituisce la risposta già ripristinata.

Usa il provider demo per provare senza chiave API. Per i provider reali (openai, anthropic, gemini, ollama) le chiavi si impostano lato server come variabili d'ambiente.

Modalità zero-knowledge

Se attivi Zero-knowledge, il server non conserva nulla: la mappa segnaposto→valore torna al tuo browser e viene usata lì per il ripristino. È l'opzione più riservata; lo svantaggio è che la mappa vive solo nella scheda corrente del browser (se ricarichi la pagina la perdi).

Le opzioni

OpzioneCosa fa
ConfidenzaSoglia 0–1: più alta = meno falsi positivi, ma rischi di non rilevare qualcosa. Default 0.7.
Tipi entitàLimita il rilevamento ad alcuni tipi (es. EMAIL,PHONE,CF). Vuoto = tutti. Accetta anche le forme estese ADDRESS/ORGANIZATION/LOCATION.
NER avanzatoAbilita il riconoscimento di nomi/organizzazioni/luoghi via modelli (se installati sul server). Più lento.
Mostra valori rilevatiNella scheda Anonimizza, elenca anche i valori originali individuati (utile per verifica).
Max chunkPipeline: lunghezza massima dei blocchi per documenti lunghi. 0 = nessuna suddivisione.
Istruzione / System promptPipeline: il prompt dato all'AI. Usa {document} dove va inserito il testo anonimizzato.

Sessioni

Ogni anonimizzazione (non zero-knowledge) crea una sessione temporanea sul server che custodisce la mappa, con scadenza automatica. L'ID sessione è una credenziale: chi lo conosce può ripristinare i dati — non condividerlo. Nella scheda Sessioni puoi verificarne lo stato o eliminarla subito.

API key dell'app

Se l'operatore ha protetto il server con una API key, inseriscila nel campo API key in alto: verrà inviata come header X-API-Key con ogni richiesta.

Tipi di dato riconosciuti

PERSON, ORG, LOC, ADDR (indirizzo), EMAIL, PHONE, IBAN, CF (codice fiscale), PIVA, CARD (carta), DATE, AMOUNT (importo), CAP, ACCOUNT (conto). I codici con checksum (IBAN, CF, P.IVA, carte) sono validati per ridurre i falsi positivi.

Per chi usa l'API direttamente

La documentazione interattiva degli endpoint è su /docs (OpenAPI/Swagger). Endpoint principali: POST /api/anonymize, POST /api/rehydrate, POST /api/process, più le varianti *-file per gli upload e GET/DELETE /api/session/{id}.
